Guided Pairing: An Introspective

Joey and Nick

What is pair programming?

Pair programming is technique for programmers to work in pairs collaboratively. One person, the driver, assumes control of the keyboard. The other, the navigator, dictates code to the driver.

Why is pair programming important to DBC?

It fosters a more collaborative environment. Dividing up the roles into driver and navigator allows one person to focus on organizing the code and allows the other person to focus their mind on creating code rather than the physical act of typing code.

Roles
Driver Navigator

  • Typing code
  • Organizing thoughts of navigator
  • Clarifying

  • Dictates code to driver
  • Researching
  • Spitballing ideas
